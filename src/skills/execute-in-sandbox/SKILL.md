# 沙箱执行工具使用说明
- 每次执行该工具时，都会在沙箱环境中创建一个全新的隔离运行空间，工具执行结束自动销毁环境。
- 执行前必须审核代码/命令内容，高危（rm -rf /、fork bomb、反弹shell、os.system("reboot") 等）→ 拒绝执行并说明危害
- 沙箱内与宿主机的文件系统是隔离的，可以通过工具函数中code_file_path将宿主机的文件或目录映射到沙箱内的`/workspace`路径下，该目录是工作目录可读写。
  例如，宿主机路径`/home/user/project`可以映射到沙箱内的`/workspace/project`，这样在沙箱内就可以访问该目录下的文件。
- 沙箱内有 /usr 只读挂载，python3/bash/gcc/g++/java 等编译器可用
- 沙箱内的pip安装，npm安装和go安装会缓存到宿主机的用户目录下，安装包和编译好的python扩展模块会持久化到宿主机的用户目录下以避免重复下载和编译。
- **记住**：沙箱内的文件操作对外部系统完全不可见。文件持久化必须通过调用dispatcher工具由 FileAgent 完成。
- 可以使用load_skill("第三方命令工具集目录")查看可以使用的第三方命令工具集。
